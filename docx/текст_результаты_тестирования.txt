Результаты тестирования и устранение ошибок

Ошибка дублирования логина пользователя
При попытке добавить пользователя с уже существующим логином возникало необработанное
исключение SQL (нарушение ограничения UNIQUE на поле username) вместо понятного сообщения
пользователю. Причина заключалась в отсутствии проверки уникальности логина на уровне
приложения. Для устранения ошибки в UserDAO был добавлен метод usernameExists(), который
вызывается в контроллере «Пользователи» перед сохранением учётной записи, а также
организован перехват исключения SQLException с выводом понятного сообщения. После исправления
при попытке добавить пользователя с дублирующимся логином пользователь видит сообщение:
«Логин «…» уже занят».

Ошибка ввода некорректного количества часов участия
При вводе в поле «Отработанные часы» нечислового значения (буквы вместо цифр) или
отрицательного числа возникала ошибка преобразования типов либо нарушение ограничения
CHECK (hours_worked >= 0). Причина – отсутствие проверки значения на уровне приложения. Для
устранения ошибки в контроллере «Участия» была добавлена валидация: введённое значение
разбирается как число и проверяется на допустимый диапазон (от 0 до 999,9). После исправления
при некорректном вводе пользователь видит сообщение «Часы — число, например 4.5» или «Часы
вне допустимого диапазона», а запись не создаётся.

Ошибка доступа к рабочим разделам для роли «Пользователь»
Пользователь с ролью USER мог видеть рабочие разделы (волонтёры, участия, журнал действий и
др.), к которым у него не должно быть доступа. Причина – отсутствие фильтрации разделов меню
по роли. Для устранения ошибки в MainController был добавлен метод проверки доступа
isAllowed(): обычному пользователю доступны только просмотровые разделы (дашборд,
мероприятия, информация о волонтёрах, рейтинг, настройки), а раздел «Пользователи» доступен
только администратору. После исправления состав бокового меню формируется строго в
соответствии с ролью.

Ошибка удаления последнего администратора
При попытке удалить или сменить роль единственного активного администратора система могла
остаться без администратора. Причина – отсутствие проверки количества активных
администраторов. Для устранения ошибки в контроллере «Пользователи» была добавлена проверка
countActiveAdmins(): удаление или разжалование последнего активного администратора
запрещается. После исправления при такой попытке пользователь видит сообщение: «Нельзя
удалить последнего активного администратора».

Ошибка валидации даты и времени мероприятия
При вводе времени мероприятия в неверном формате возникала ошибка преобразования
(DateTimeParseException). Причина – отсутствие проверки формата ввода. Для устранения ошибки
в контроллере «Мероприятия» была добавлена обработка: время разбирается по шаблону HH:mm, и
при ошибке выводится понятное сообщение. После исправления при неверном формате пользователь
видит сообщение «Время в формате HH:mm.», а мероприятие не сохраняется.

Ошибка отображения кириллицы при экспорте в CSV
При открытии выгруженного CSV-файла в Microsoft Excel русский текст отображался в виде
«кракозябр» (неверная кодировка). Причина – отсутствие маркера кодировки UTF-8 (BOM) в начале
файла. Для устранения ошибки в утилите CsvUtil в начало файла был добавлен BOM, а сам файл
записывается в кодировке UTF-8. После исправления русский текст во всех выгружаемых
CSV-файлах (волонтёры, рейтинг, журнал действий) отображается корректно.

Ошибка удаления волонтёра без предупреждения о связанных участиях
При удалении волонтёра вместе с ним каскадно удалялись все его записи участия
(ON DELETE CASCADE), однако пользователь не предупреждался об этом. Причина – отсутствие
подтверждения перед удалением связанных данных. Для устранения ошибки в контроллере
«Волонтёры» было добавлено диалоговое подтверждение действия. После исправления при удалении
волонтёра пользователю выводится запрос: «Удалить волонтёра «…»? Все его участия тоже будут
удалены», и удаление выполняется только после согласия.

Итоговый вывод. По результатам тестирования все 11 тест-кейсов были успешно пройдены.
Выявленные проблемы устранены. Программный продукт «VolonterHoursAPP 1.0» готов к
эксплуатации и соответствует заявленным требованиям.
